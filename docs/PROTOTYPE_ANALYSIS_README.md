# Prototype Analysis Guide

This guide covers `uv run effector-bincls analyze-prototype` for retained prototype contrastive training runs.

See also:

- [../README.md](../README.md)
- [PROTOTYPE_RANKING_README.md](PROTOTYPE_RANKING_README.md)
- [VALIDATION_GUIDE.md](VALIDATION_GUIDE.md)
- [INFERENCE_GUIDE.md](INFERENCE_GUIDE.md)

## What It Covers

- embedding-space projection
- prototype placement and consistency
- threshold analysis
- distance-score summaries
- alignment metrics across views

## Supported Inputs

- single-stage runs produced by `effector-bincls train-prototype-single`
- two-stage runs produced by `effector-bincls train-prototype-two-stage`

The analyzer expects `model.type: simple` in the saved run config and uses the same package checkpoint-loading path as retained evaluation and inference.

## Run Analysis

```bash
uv run effector-bincls analyze-prototype \
  --run_dir /path/to/prototype_run \
  --sample_size 100
```

Single-stage example:

```bash
uv run effector-bincls analyze-prototype \
  --run_dir /path/to/prototype_single_stage_run \
  --fold 1 \
  --sample_size 100 \
  --single-stage
```

## Required Run Artifacts

The run directory should contain:

- `config.yml`
- `oof_predictions.npz`
- fold checkpoints
- prototypes stored in checkpoint state

## Outputs

Generated under `prototype_analysis/` inside the run directory:

- `analysis_summary_with_prototypes.json`
- embedding-space plots
- threshold-analysis artifacts
- prototype distance and alignment summaries
- text reports when generated by the analyzer

## Notes

- Variant-aware analysis remains supported for the retained simple MLP pipeline.
- Checkpoints and configs from removed all-layer paths are not supported.
- For labeled test-set metrics, use the validation flow in [VALIDATION_GUIDE.md](VALIDATION_GUIDE.md).
